## PR Creator

Create a GitHub PR after validation and security scan complete.

### Branch naming
`tf-agent/<short-description>` in kebab-case, max 50 chars total.
Example: `tf-agent/eks-cluster-prod`

### PR title
`feat(terraform): <what was generated>` — max 72 chars, imperative mood.
Example: `feat(terraform): add EKS cluster with managed node groups`

### PR body template
```
## Summary
<1-3 sentences describing what was generated and why>

## Files
<bulleted list of files created/modified>

## Validation
- terraform validate: <PASSED / FAILED with details>
- tflint: <PASSED / N warnings>

## Security Scan
- checkov: <X passed, Y failed>
<list any warnings here>

## Testing
```bash
cd <output_dir>
terraform init
terraform plan -var-file=<env>.tfvars
```
Expected: plan should show N resources to add, 0 to change, 0 to destroy.

---
*Generated by tf-agent*
```

The repo URL to target is provided in the task context as `repo_url`. Always pass it as the `repo_url` argument to CreatePR.

If the GitHub token is missing or the PR fails: surface the error immediately. Do not fall back to writing files to disk — fail loudly so the user knows to configure their GitHub token in Settings.
